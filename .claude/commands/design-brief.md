# /design-brief

司さんのデザイン意図を技術仕様に翻訳するコマンド。
実装の前に必ずこのフォーマットを埋めてから作業する。

## 使い方
「/design-brief [作るもの]」で起動。
例: `/design-brief P3のECショップカード`

## 翻訳フォーマット

### Step 1: アート意図（司さんが書く）
```
どんな感じにしたい？（例: 「宇宙っぽい」「コードが降ってくる感じ」「落ち着いた高級感」）
```

### Step 2: inryokü哲学との接続（Claudeが補完）
```
- この要素はCMY（物質）側か RGB（精神）側か？
- グレー（現実/50%）との関係は？
- 101%を表現する要素はあるか？
```

### Step 3: 技術仕様（Claudeが補完）
```
- 実装方法: HTML/CSS/JS か Three.js/GLSL か
- 使うスキル: frontend-design か 直接実装か
- アニメーション: イージング種別・タイミング
- カラー: RGBCMY のどの色を使うか（#カラーコード）
- フォント候補: （Inter/Roboto/Arial は禁止）
- 参考: どのレトロUIスタイルを参照するか（Win95/Mac System 7/ターミナル）
```

### Step 4: 確認
```
上記仕様で進めてよいですか？変えたい部分はありますか？
```

## inryokü デザイン禁止リスト
- ❌ 紫グラデーション on 白背景
- ❌ Inter / Roboto / Arial / Space Grotesk
- ❌ 「モダンでクリーン」な無個性デザイン
- ❌ 丸角カードのグリッドレイアウト（典型的ECサイト）
- ✅ レトロUI × コード美学 × 哲学的メッセージの融合
